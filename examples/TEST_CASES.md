# STAgent 测试用例分类说明

本文档详细说明了 STAgent 项目中所有测试用例的分类、功能和测试要点。

## 测试用例总览

| 分类 | 数量 | 测试程序 |
|------|------|----------|
| 基础测试 | 4 | hello, echo, counter, compare |
| 数学计算 | 6 | add, divide, factorial, gcd, prime, sort |
| 字符串处理 | 4 | reverse, case, strlen, palindrome |
| 文件IO | 3 | cat, write, wc |
| 错误处理 | 2 | validate, memory |
| **总计** | **19** | |

## 1. 基础测试 (Basic Tests)

### 1.1 hello.exe
**功能**: 打印问候语
- **输入**: 可选的名字参数
- **输出**: "Hello, [name]!" 或 "Hello, World!"
- **测试点**: 
  - 无参数时输出默认问候
  - 有参数时输出自定义问候
  - 输出包含 "Hello"
  - 退出码为 0

### 1.2 echo.exe
**功能**: 回显输入的文本
- **输入**: 一个或多个文本参数
- **输出**: 原样输出所有参数（空格分隔）
- **测试点**:
  - 正确回显单个参数
  - 正确回显多个参数
  - 处理空字符串
  - 无参数时返回错误

### 1.3 counter.exe
**功能**: 从起始数字计数到结束数字
- **输入**: start, end (两个整数)
- **输出**: 逐行打印从 start 到 end 的所有数字
- **测试点**:
  - 正确的计数序列
  - start > end 时报错
  - 参数数量验证

### 1.4 compare.exe
**功能**: 比较两个数字的大小
- **输入**: num1, num2 (两个整数)
- **输出**: GREATER / LESS / EQUAL
- **测试点**:
  - 正确判断大于关系
  - 正确判断小于关系
  - 正确判断相等关系
  - 输出格式验证

## 2. 数学计算 (Math Tests)

### 2.1 add.exe
**功能**: 两数相加
- **输入**: num1, num2 (两个浮点数)
- **输出**: 和（保留两位小数）
- **测试点**:
  - 正数加法
  - 负数加法
  - 小数加法
  - 输出格式（两位小数）

### 2.2 divide.exe
**功能**: 两数相除
- **输入**: dividend, divisor (被除数、除数)
- **输出**: 商（保留两位小数）
- **测试点**:
  - 正常除法
  - 除数为零时报错（退出码 2）
  - 负数除法
  - 错误信息输出到 stderr

### 2.3 factorial.exe
**功能**: 计算阶乘
- **输入**: number (0-20 的整数)
- **输出**: 阶乘结果
- **测试点**:
  - 0! = 1
  - 正整数阶乘
  - 超出范围时报错
  - 负数时报错

### 2.4 gcd.exe
**功能**: 计算最大公约数
- **输入**: num1, num2 (两个正整数)
- **输出**: 最大公约数
- **测试点**:
  - 互质数的 GCD
  - 有公约数的情况
  - 相同数字的 GCD
  - 非正数时报错

### 2.5 prime.exe
**功能**: 判断是否为素数
- **输入**: number (非负整数)
- **输出**: YES / NO
- **测试点**:
  - 素数判断（2, 3, 5, 7, 11...）
  - 合数判断（4, 6, 8, 9...）
  - 边界情况（0, 1）
  - 退出码（素数返回 0，合数返回 1）

### 2.6 sort.exe
**功能**: 对输入的整数进行排序
- **输入**: 多行整数（每行一个数字）
- **输出**: 排序后的结果，每行一个数字
- **测试点**:
  - 正整数排序
  - 负整数排序
  - 混合正负整数排序
  - 重复数字处理
  - 单个数字处理
  - 升序输出验证

## 3. 字符串处理 (String Tests)

### 3.1 reverse.exe
**功能**: 反转字符串
- **输入**: text (字符串)
- **输出**: 反转后的字符串
- **测试点**:
  - 普通字符串反转
  - 空字符串处理
  - 数字字符串反转
  - 特殊字符处理

### 3.2 case.exe
**功能**: 大小写转换
- **输入**: mode (upper/lower), text (字符串)
- **输出**: 转换后的字符串
- **测试点**:
  - 转大写
  - 转小写
  - 无效模式时报错
  - 混合大小写输入

### 3.3 strlen.exe
**功能**: 统计字符串信息
- **输入**: text (字符串)
- **输出**: 总长度、字母数、数字数、空格数、其他字符数
- **测试点**:
  - 纯字母统计
  - 混合字符统计
  - 空字符串处理
  - 输出格式验证

### 3.4 palindrome.exe
**功能**: 判断是否为回文
- **输入**: text (字符串)
- **输出**: YES / NO
- **测试点**:
  - 回文字符串（racecar, madam）
  - 非回文字符串
  - 单字符处理
  - 大小写不敏感

## 4. 文件IO (File I/O Tests)

### 4.1 cat.exe
**功能**: 读取并显示文件内容
- **输入**: filename (文件路径)
- **输出**: 文件内容
- **测试点**:
  - 成功读取存在的文件
  - 文件不存在时报错（退出码 2）
  - 错误信息输出到 stderr
  - 多行文件处理

### 4.2 write.exe
**功能**: 写入内容到文件
- **输入**: filename (文件路径), content (内容)
- **输出**: 成功消息
- **测试点**:
  - 成功创建并写入文件
  - 覆盖已存在文件
  - 无法创建文件时报错
  - 确认消息输出

### 4.3 wc.exe
**功能**: 统计文件的行数、单词数、字符数
- **输入**: filename (文件路径)
- **输出**: Lines, Words, Chars 统计
- **测试点**:
  - 正确统计行数
  - 正确统计单词数
  - 正确统计字符数
  - 文件不存在时报错

## 5. 错误处理 (Error Handling Tests)

### 5.1 validate.exe
**功能**: 参数验证（数字范围检查）
- **输入**: number (整数)
- **输出**: 验证结果或错误信息
- **测试点**:
  - 有效范围内的数字（0-100）
  - 负数时报错（退出码 2）
  - 超过 100 时报错（退出码 3）
  - 参数数量错误（退出码 1）
  - 不同退出码的区分

### 5.2 memory.exe
**功能**: 内存分配测试
- **输入**: size_mb (内存大小，MB)
- **输出**: 分配成功或失败信息
- **测试点**:
  - 小内存分配成功
  - 负数时报错（退出码 2）
  - 超大内存时报错（退出码 3）
  - 分配失败时报错（退出码 4）
  - 多种错误退出码

## 测试配置说明

每个测试程序都有对应的 YAML 配置文件，包含：

1. **target**: 目标程序路径和超时设置
2. **wrapper**: 参数生成规则
   - `type`: int, float, string, list
   - `range`: 数值范围
   - `values`: 枚举值列表
3. **assertions**: 断言规则
   - `exact`: 精确匹配
   - `fuzzy`: 模糊匹配
   - `regex`: 正则表达式
   - `contains`: 包含检查
   - `numeric_range`: 数值范围
   - `exit_code`: 退出码检查
   - `no_error`: 无错误检查

## 使用方法

### CLI 模式
```bash
# 运行单个测试
python -m stagent.cli --config examples/basic/hello.yaml

# 运行分类测试
python -m stagent.cli --config examples/math/add.yaml
```

### Web 模式
1. 启动服务: `cd stagent-web && start.bat`
2. 访问: http://localhost:5173
3. 选择配置文件并运行测试
