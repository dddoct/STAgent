# STAgent 前端方案

## 一、技术选型

### 推荐方案：React + TailwindCSS

| 技术 | 选择 | 理由 |
|------|------|------|
| 框架 | React 18 | 生态丰富，组件化成熟 |
| UI库 | TailwindCSS + HeadlessUI | 快速开发，样式灵活 |
| 状态管理 | Zustand | 轻量，够用 |
| 图表 | Recharts | 覆盖率可视化 |
| 构建 | Vite | 快速启动 |

### 备选方案

| 方案 | 适用场景 |
|------|----------|
| Vue 3 + Element Plus | 团队熟悉Vue |
| Next.js | 需要SSR |
| 纯HTML + Vanilla JS | 极简需求 |

## 二、功能模块

```
┌─────────────────────────────────────────────────────────────────┐
│                         STAgent Web                              │
├─────────────────────────────────────────────────────────────────┤
│  ┌────────────┐  ┌────────────┐  ┌────────────┐  ┌────────────┐  │
│  │   配置     │  │   测试     │  │   报告     │  │   覆盖     │  │
│  │   管理     │  │   运行     │  │   查看     │  │   率       │  │
│  └────────────┘  └────────────┘  └────────────┘  └────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 模块0：登录认证
- 登录/注册表单（Tab 切换）
- JWT Token 自动管理
- 游客模式（无需登录直接访问）
- 登录状态持久化（localStorage）
- 401 自动跳转登录

### 模块1：配置管理
- YAML 配置编辑器（ Monaco Editor）
- Wrapper 可视化配置
- 预设用例管理
- 一键导入/导出配置

### 模块2：测试运行
- 实时进度显示
- 动态日志输出
- 暂停/停止控制
- 用例执行状态（通过/失败/错误）

### 模块3：报告查看
- 测试结果表格
- 断言详情折叠
- 快速筛选（只看失败）
- 导出报告（JSON/HTML/PDF）

### 模块4：覆盖率可视化
- 代码覆盖率仪表盘
- 行覆盖热力图
- 函数覆盖列表
- 未覆盖代码高亮

## 四、技术架构

### 4.1 前后端交互

```
┌─────────────────┐          ┌─────────────────┐
│   React Front   │  <───>   │   Python API    │
│   (端口 3000)    │  REST    │   (端口 8000)    │
└─────────────────┘          └─────────────────┘
                                    │
                                    ▼
                            ┌─────────────────┐
                            │   STAgent      │
                            │   Core Engine  │
                            └─────────────────┘
```

### 4.2 API 设计

**认证 API**

| 接口 | 方法 | 说明 |
|------|------|------|
| `POST /api/auth/register` | POST | 用户注册 |
| `POST /api/auth/login` | POST | 用户登录，返回 JWT token |
| `GET /api/auth/me` | GET | 获取当前用户信息 |
| `POST /api/auth/logout` | POST | 登出 |

**业务 API**

| 接口 | 方法 | 说明 |
|------|------|------|
| `GET /api/projects` | GET | 获取项目列表 |
| `POST /api/projects` | POST | 创建项目 |
| `GET /api/projects/{id}` | GET | 获取项目详情 |
| `PUT /api/projects/{id}` | PUT | 更新项目 |
| `DELETE /api/projects/{id}` | DELETE | 删除项目 |
| `POST /api/run` | POST | 运行测试 |
| `GET /api/run/{task_id}` | GET | 获取任务状态 |
| `POST /api/run/{task_id}/stop` | POST | 停止测试 |
| `GET /api/run/{task_id}/results` | GET | 获取测试结果 |
| `GET /api/reports/{id}` | GET | 获取报告 |
| `GET /api/reports/task/{task_id}` | GET | 根据任务ID获取报告 |
| `GET /api/coverage/{id}` | GET | 获取覆盖率 |
| `POST /api/upload/source` | POST | 上传源码 |
| `POST /api/upload/binary` | POST | 上传可执行文件 |

### 4.3 WebSocket 实时推送

```javascript
// 实时日志推送
ws.on('log', (data) => {
  console.log(data); // {level: 'info', message: '...'}
});

// 测试进度推送
ws.on('progress', (data) => {
  setProgress(data.current, data.total);
});

// 测试结果推送
ws.on('result', (data) => {
  addResult(data);
});
```

## 七、替代方案：极简版

如果不想搭建完整前端，可以使用 **Python 内置 HTTP 服务器 + 简单 HTML**：

```
stagent/
├── web/
│   ├── index.html      # 单页应用
│   ├── style.css       # 样式
│   └── app.js          # 逻辑
└── server.py           # Flask/FastAPI 后端
```

**适合场景**：快速验证、小团队内部使用

## 八、优先级建议

| 阶段 | 功能 | 复杂度 |
|------|------|--------|
| P0 | 配置编辑器 + 运行测试 | 中 |
| P0 | 测试结果展示 | 低 |
| P1 | 覆盖率仪表盘 | 中 |
| P1 | 实时日志 | 低 |
| P2 | 多项目支持 | 中 |
| P2 | 报告导出 | 低 |

**建议先做 P0 功能，用极简版验证，再逐步完善。**
